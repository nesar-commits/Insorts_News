# Insorts News

A full-stack news aggregator: real public RSS feeds ingested into Postgres, served by a
FastAPI backend, and rendered by a React app that ships as a responsive website **and**
as native iOS/Android apps (via Capacitor).

## Stack

| Layer    | Tech |
|----------|------|
| Backend  | Python, FastAPI, SQLAlchemy 2.0, Alembic, PostgreSQL, APScheduler, feedparser, JWT auth |
| Frontend | React 19, Vite, React Router, TanStack Query, Tailwind CSS v4, Axios |
| Mobile   | Capacitor (wraps the React build into real Xcode / Android Studio projects) |

## How it works

- A background job (APScheduler, inside the FastAPI process) polls ~15 real RSS feeds
  (BBC, TechCrunch, The Verge, ESPN, NPR, Al Jazeera, NASA, Variety, ...) every 15
  minutes, dedupes by article URL, and stores new articles in Postgres.
- FastAPI exposes a REST API for categories, paginated/searchable articles, trending
  headlines, JWT auth (register/login), and per-user bookmarks.
- The React app is one codebase for both web and mobile: category feed with infinite
  scroll, search, article detail, auth, saved articles, dark mode, and a mobile bottom
  tab bar. Capacitor packages that same build into native app shells for iOS and
  Android.

## Project layout

```
backend/
  app/
    core/        # settings, JWT/password hashing
    db/          # SQLAlchemy engine/session/base
    models/      # User, Category, Source, Article, Bookmark
    schemas/     # Pydantic request/response models
    crud/        # DB query functions
    api/routes/  # auth, users, categories, articles, bookmarks
    services/    # RSS source list, ingestion logic, APScheduler wiring
  alembic/       # migrations
  scripts/seed.py

frontend/
  src/
    api/         # axios client + endpoint wrappers
    context/     # Auth, Theme, Toast
    hooks/       # infinite scroll, debounce, bookmark mutation
    components/  # NavBar, BottomNav, ArticleCard, HeroCarousel, ...
    pages/       # Feed, SearchPage, ArticleDetail, Bookmarks, Login, Register, Profile
  ios/           # native Xcode project (generated by Capacitor)
  android/       # native Android Studio project (generated by Capacitor)
  resources/     # source icon.svg/splash.svg used to generate all app icons
```

## Prerequisites

- Python 3.12, Node.js 20+, PostgreSQL (already running locally on this machine as a
  Homebrew service).
- To actually **build/run the native apps**: Xcode + CocoaPods (iOS, macOS only) and
  Android Studio (Android). Not required to run the web app.

## Backend setup

```bash
cd backend
python3.12 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

createdb insorts_news                 # first time only
cp .env.example .env                  # then edit JWT_SECRET_KEY, DATABASE_URL
alembic upgrade head                  # create schema
python -m scripts.seed                # seed categories/sources + first RSS pull

uvicorn app.main:app --reload --port 8000
```

The API is now at `http://localhost:8000/api` (health check: `/api/health`). The RSS
ingestion job also runs automatically every `RSS_FETCH_INTERVAL_MINUTES` while the
server is up.

## Frontend setup (web)

```bash
cd frontend
npm install
npm run dev          # http://localhost:5173, talks to the backend via VITE_API_BASE_URL
```

`frontend/.env` controls the API base URL (`VITE_API_BASE_URL`). For production, build
with `npm run build` and serve `frontend/dist` behind any static host / reverse proxy
that points `/api` at the deployed FastAPI service.

## Mobile apps (iOS + Android)

The `ios/` and `android/` folders are already generated and synced with a production
web build, including app name, icon, and splash screen. Whenever frontend code
changes, rebuild and re-sync before opening the native project:

```bash
cd frontend
npm run build
npx cap sync
```

**Before testing on a real device or emulator**, point the app at a backend it can
actually reach — `localhost` in `.env` only resolves on this machine:

- Android emulator → backend on the host machine: `http://10.0.2.2:8000/api`
- Physical device / iOS simulator → your machine's LAN IP: `http://192.168.x.x:8000/api`
- Production → your deployed HTTPS API URL

Update `VITE_API_BASE_URL` accordingly, `npm run build`, then `npx cap sync` again.

### iOS

```bash
npx cap open ios
```

Opens the project in Xcode. Select a simulator or device and hit Run. To ship:
set your Team/signing in the App target, archive (`Product > Archive`), and upload via
Xcode Organizer to App Store Connect (requires an active Apple Developer Program
membership).

### Android

```bash
npx cap open android
```

Opens the project in Android Studio. Run on an emulator/device directly. To ship:
`Build > Generate Signed Bundle/APK`, create/select a keystore, and upload the `.aab`
to the Play Console (requires a Google Play Developer account).

## Environment variables

**backend/.env**
| Var | Purpose |
|-----|---------|
| `DATABASE_URL` | Postgres connection string |
| `JWT_SECRET_KEY` | Signs auth tokens — set a real random secret in production |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | JWT lifetime |
| `CORS_ORIGINS` | Comma-separated allowed origins (includes `capacitor://localhost` for the mobile shell) |
| `RSS_FETCH_INTERVAL_MINUTES` | How often the background ingest job runs |

**frontend/.env**
| Var | Purpose |
|-----|---------|
| `VITE_API_BASE_URL` | Base URL the app calls for all API requests |

## Known limitations

- A handful of sources (ESPN, TechCrunch, Al Jazeera) don't embed any image in their
  RSS feed at all — those articles show a branded gradient card with the category icon
  instead of a photo, by design, rather than a broken image.
- Native binaries (`.ipa`/`.apk`) were not compiled in this environment (no Xcode.app /
  Android Studio installed here) — the native projects are fully generated and synced,
  ready to open and build on a machine with those toolchains installed.
